# ⚠️ اقرأ قبل الرفع — خطوات الإصلاح

## اللي حصل بالظبط
ملف `menu-data.json` اللي على GitHub اتحول لقيمة فاضية (`null`) بعد نشر سابق. ده سبب كل الأعراض: الحقول فاضية، المعاينة فاضية، ومفيش حفظ بيحصل.

## اللي اتعمل في الملفات دي
1. **index.html** (لوحة التحكم): اتضاف كود حماية يمنع:
   - تحميل بيانات فاضية بصمت
   - حفظ بيانات فاضية في المتصفح
   - **نشر بيانات فاضية على GitHub** (خط الدفاع الأهم)
   لو حصل نفس الموقف تاني، هتشوف رسالة تحذير واضحة بدل ما الحقول تفضل فاضية من غير تفسير.

2. **menu.html** (صفحة المنيو): اتضاف فحص لو الملف رجع قيمة غير صالحة، يوريك رسالة "ملف المنيو غير صالح" بدل ما يفضل فاضي بصمت.

3. **menu-data.json**: ملف جديد نظيف، فيه:
   - اسمك ورقم الواتساب زي ما كانوا (تياترو مصر كافيه، 01126023106)
   - أصناف تجريبية (زي الأول) تقدر تبني عليها أقسامك الحقيقية من جديد

## خطوات الرفع (بالترتيب)

1. روح لريبو GitHub بتاعك: `github.com/coffe3883-ui/Teatro-Masr-Cafe`
2. ارفع الـ 3 ملفات دي (`index.html`, `menu.html`, `menu-data.json`) بدل القديمين — استخدم "Add file" → "Upload files"، واختار الملفات، ولما يسألك اكتب أي رسالة زي "إصلاح مشكلة البيانات الفارغة" ودوس Commit
3. **افتح لوحة التحكم في نافذة متصفح جديدة تمامًا (أو خاصة/Incognito)** — ده مهم جدًا، عشان تتأكد إن المتصفح مش قاعد يقرا نسخة قديمة محفوظة عنده (Cache)
4. سجل دخول، اضبط إعدادات الربط مع GitHub من جديد (نفس الـ Token اللي اشتغل معاك)
5. جرب تضيف بيانات كافيتك الحقيقية من جديد، وانشر، وتأكد من المعاينة

## نصيحة لتجنب تكرار المشكلة
بعد كل نشر مهم، افتح رابط المنيو `https://coffe3883-ui.github.io/Teatro-Masr-Cafe/menu.html` في تاب جديد وتأكد إن البيانات ظاهرة صح، قبل ما تكمل تعديلات تانية فوقها.
